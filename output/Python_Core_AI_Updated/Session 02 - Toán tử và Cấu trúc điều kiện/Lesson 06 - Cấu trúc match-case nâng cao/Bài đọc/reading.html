1
Đặt vấn đề

Khi làm việc với các phản hồi từ API hoặc cấu trúc dữ liệu phức tạp dạng từ điển (Dictionary) và danh sách lồng nhau, việc sử dụng các khối lệnh if-elif-else truyền thống để kiểm tra sự tồn tại của khóa, kiểm tra kiểu dữ liệu của từng giá trị và trích xuất dữ liệu thường đòi hỏi viết rất nhiều mã nguồn lặp đi lặp lại. Điều này dẫn đến cấu trúc code cồng kềnh, dễ phát sinh lỗi logic và rất khó để bảo trì.

2
Phân tích Bản chất

Cấu trúc match-case (từ Python 3.10+) mang lại giải pháp Structural Pattern Matching mạnh mẽ. Thay vì chỉ kiểm tra giá trị, match-case cho phép kiểm tra cấu trúc của dữ liệu. Mapping Pattern giúp so khớp từ điển dựa trên sự hiện diện của các khóa mà không cần kiểm tra thủ công từng khóa. Capture Pattern sử dụng từ khóa 'as' để vừa kiểm tra cấu trúc vừa gán một phần hoặc toàn bộ dữ liệu khớp vào một biến để sử dụng trực tiếp trong khối xử lý.

3
Giới thiệu Giải pháp

Cách triển khai tối ưu sử dụng match-case kết hợp Mapping Pattern để kiểm tra các khóa của từ điển và Capture Pattern ('as') để bắt giữ dữ liệu.

Sơ đồ luồng match-case:

Input -> [Match Pattern?] -> (True) -> [Guard Constraint?] -> (True) -> Execute

-> (False) ---------> [Next Pattern] ...

-> Không khớp -------> Wildcard (case _)

4
Mã nguồn chuẩn hóa & Phân tích thực thi
Nhấp vào để đóng/mở mã nguồn chi tiết
example.python 
def parse_api_response(response):
    match response:
        case {"status": "success", "data": {"user": str(name), "role": role}}:
            return f"User: {name}, Role: {role}"
        case {"status": "error", "error": {"code": int(code), "message": msg}}:
            return f"Error [{code}]: {msg}"
        case {"status": _} as raw_response:
            return f"Unknown status: {raw_response['status']}"
        case _:
            return "Invalid response format"

Hàm xử lý sẽ nhận vào đầu vào là một từ điển. Tại case đầu tiên, Python sử dụng Mapping Pattern để kiểm tra xem từ điển có các khóa 'status' bằng 'success' và 'data' chứa 'user' (phải là kiểu chuỗi) và 'role' hay không, nếu đúng sẽ tự động trích xuất các biến này. Ở case thứ ba, Capture Pattern 'as' được áp dụng để giữ lại toàn bộ đối tượng từ điển vào biến 'raw_response' khi phát hiện có khóa 'status' bất kỳ. Case cuối cùng '_' là wildcard để bắt mọi cấu trúc không hợp lệ còn lại.

LƯU Ý QUAN TRỌNG

Cấu trúc match-case nâng cao giúp đơn giản hóa việc giải nén và kiểm tra dữ liệu phức tạp thông qua Mapping Pattern và Capture Pattern. Ba lỗi thường gặp cần tránh:

  1. Đặt wildcard case _ sai vị trí ở trước các case cụ thể làm che khuất và khiến các case phía sau không bao giờ được thực thi.
  2. Hiểu lầm về Mapping Pattern khi nghĩ nó yêu cầu khớp chính xác tuyệt đối mọi key; thực tế, pattern chỉ cần khớp tối thiểu các key được khai báo trong case.
  3. Cú pháp gán biến sai trong capture pattern, ví dụ dùng dấu gán '=' thay vì từ khóa 'as' để ràng buộc dữ liệu vào định danh mới.
